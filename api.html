LLM on a Stick API

Our flagship language model endpoint. Trained on a diverse dataset including Stack Overflow, cookbook recipes, and whatever was trending on Twitter in 2019.

POST /v1/completions

Request Body

Parameter Type Description
prompt string What you want the model to do (required)
max_tokens integer How long the response should be (1-2048)
temperature float Creativity level (0.0-2.0). Higher = weirder.
confidence string How confident you want us to sound ("low", "medium", "enterprise")

Example Request

{
  "prompt": "Write a professional email apologizing for missing a deadline",
  "max_tokens": 200,
  "temperature": 0.7,
  "confidence": "enterprise"
}

Example Response

{
  "id": "comp_beach_42069",
  "object": "completion",
  "created": 1693526400,
  "model": "llm-stick-v2",
  "choices": [{
    "text": "Dear [Name],\\n\\nI hope this message finds you well...",
    "finish_reason": "completed",
    "confidence_score": 0.73
  }],
  "usage": {
    "prompt_tokens": 12,
    "completion_tokens": 67,
    "seashells_consumed": 3
  }
}
üí° Pro Tip: Setting confidence: "enterprise" doesn't change the model output, but it does make the response 30% longer and includes phrases like "strategic alignment" and "synergistic outcomes."

Agentic Seagull API

Autonomous task orchestration powered by reinforcement learning and french fries.

POST /v1/seagull/execute

How It Works

  1. Submit a high-level goal
  2. The Seagull breaks it into subtasks
  3. Tasks are executed by an agent swarm
  4. Results are aggregated (sometimes)
  5. You get notified when something happens

Request Body

Parameter Type Description
goal string What you want accomplished
tools array Available tools ("web_search", "email", "steal_fries")
persistence integer How many retries before giving up (1-10)

Example Request

{
  "goal": "Find competitor pricing and draft comparison report",
  "tools": ["web_search", "email"],
  "persistence": 7
}

Example Response

{
  "task_id": "seagull_dive_8291",
  "status": "in_progress",
  "subtasks": [
    {
      "id": "subtask_1",
      "action": "search_web",
      "status": "completed",
      "attempts": 1
    },
    {
      "id": "subtask_2",
      "action": "analyze_results",
      "status": "in_progress",
      "attempts": 3,
      "note": "Agent distracted by sandwich"
    }
  ],
  "estimated_completion": "2-6 business tides"
}
‚ö†Ô∏è Known Issues: Agents may occasionally prioritize sandwich acquisition over task completion. This is considered a feature, not a bug. It demonstrates adaptive priority management.

Error Codes

Code Status Meaning
200 OK It worked! High five.
400 Bad Request Your request was bad and you should feel bad
401 Unauthorized API key missing or notebook got wet
429 Too Many Requests Slow down, you're making the seagulls anxious
451 Tide's Out Server is underwater. Try again in 6-12 hours.
500 Internal Error Something broke. Joe is on it (probably)
503 Seagull Distracted Agent swarm busy with sandwich. Retry in 5min.
